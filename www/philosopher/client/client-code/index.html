Evolution of new modules

Start life in specific app => extract to common patterns => create npm module package

Use configuration files to pull apps together from a build system and js or css manifest system of your desires. The key is modularity. Only bespoke code that is specific to the problem needs to live in the project directory, otherwise code can be pulled in from common patterns

Apps could be built from anything that can be browserified so ampersand and mercury would be ideal candidates.

A StackMate bundle consists of

  • build config
  • js manifest
  • css manifest

See it-starts-upfront for an example of a StackMate bundle that creates a static site using metalsmith, along with two angular based apps for mobile and line of business.
