Observations

Although there is no subdomain for the blog this would be quite typical and although Drifty does not have mobile and desktop apps for members — that I know of — this would be a typical requirement of most startups.

  • The Ionic brand is consistent in all they communicate
  • Multiple build systems and CSS theme configurations
  • Heavy javascript and Node bias but also use Ruby and Rails for their soluton stack

Given that Ionic Framework respresents a fair representation of a solution stack excluding the demo apps and the forum software, a conservative count shows 7 build configs, CSS and JS stacks to maintains. If you factor in the need for specific marketing campaign pages or facebook pages this number could easily grow.

Now considering that you would to maintain a clearly seperated project structure for each sub-domain. If you apply this analysis to my problem and think about the following questions you will begin to understand why StackMates exists.

  • How much of this code is common?
  • How much effort to align 147 config files for build, css and javascript?
  • How much effort to align consistent branding?
  • How much effort to reflect improved CSS and UX Patterns?
  • How to maintain a consistent style across projects?
  • How much time researching NPM and other package.json for working modules?
  • How to get those modules working with browserify including testing Angular?
  • How can I quickly add to reference for documentation as I learn for when I forget how things work later?
  • Wouldn’t it be more profitable to have the whole solution stack in JS? forum, ecommerce
  • How could markdown content from static websites be reused in apps?
  • Is it possible to provide a more engaging UX experience with this content in apps?
  • How can youu do more than just star interesting projects on github?
  • How can Programmer Anarchy and Lean Startup principles be applied?
  • How do you maintain evolving best practices across all projects?
  • What does a concrete example of a micro-service architectured solution look like?
  • How docker smooth the workflow experience from development to production for MSA?
  • Could you engineer a develop process that contantly evolves your startup’s sales pitch as a side effect?

I’ve worked on direct marketing and lived through the painful inefficiency of replicating a home sites styles for a new landing page campaign so I know there is value there.

I rewatch talks on Programmer Anarchy and Micro-services by Fred George from outpace.com regularly and although I don’t understand all the details from my past experience with billing system and routing logic in telecoms, I can imagine an architecture that will power the business model of the future.

StackMates

You can create a static website like this one a desktop web and a mobile app with CORS authentication connected to micro-service backend in around 20 mins plus the time to have a coffee while npm install runs.

For your next project you can just copy the entire content of an existing project, edit some config files, change a few requirements and you have a whole new three stacked solution in about 15 minutes without running npm install. Want to add a new sub domain to a new project that is even easier.

So what are StackMates?

StackMates are built up from the following configuration files…

  • NPM package.json
  • Rework main.css or SASS main.scss
  • Browserify index.js
  • Gulp config
  • Hapi config
  • Dockerize config??? - fig, nscale

The idea is that you could share these StackMates configuration files that would work inside a common engine that would comprise of code in flux from going app specific to being extracted into independent modules. A side benefit would be that stacks would be easier to compare and analyse to see what trends were developing for the purposes of deciding how to invest efforts in education.

Philosophy

Node.js is the central glue layer

The number one goal is to provide concrete examples for how this rapidly expanding micro-culture of web components can be pulled together compare and learn from different strategies.

This engine is largely upon Gulp and NPM and Browerify and Senecajs the glue between components, but it would be pretty sweet to see a completely different engine and strategy to do the same thing using duo and an alternative build system with graftjs for example.

Adopt the lingua franca

Part of the Dreamineering + StackMates philosophy is delay choosing software solutions until you really know your use case and that most small projects don’t need project management software as you have all you need to communicate goals and track important information in google drive.

The intent is tightly integrate a basic CMS solution with business operations to provide greater process efficiencies.

If you work with non technical users they can create content with stackedit.io which integrates with google+ to upload images, which gives provides a nice opportunity to streamline your workflow. That is the theory at least, unfortunately stackedit.io is too sluggish for me.

This project starter directory has been setup with template files help drive and communicate how your project evolves from the creative process though to providing current status reports using various free tools and the lingua franca of business spreadsheets. The benefit here is that no-one needs to learn a new tool to understand how the project is evolving.
