Right now you create multiple client sites or apps from a starter project to build a full suite of properties that maximise code reuse with CORS authentication, connected to backend architecture that can be run as a single app or micro-service architecture. All in around 20 mins plus the time to have a coffee while npm install runs.

StackMates consist of a collection of configuration pull together fullstack solutions from micro components. The creation of StackMates was directly influenced by watching the following talks and the thought that it would great to have a better way to share ideas for how micro components can be creatively combined to solve larger problems.

  • Client: Tiny Modules - Substack
  • Server: Implementing Microservice Architectures - Fred George

Theory & goals

  • Develop a master build engine
  • Identify a solid core of standard modules by job to be done
  • Identify and create taxonomy for standard UX patterns
  • Don’t put build logic in your app specific code
  • Create a combined spotter network for fullstack
  • Developers would share StackMate bundles
  • Only bespoke domain logic lives in github project
  • Develop a consistent evolution path for new npm modules or services
  • Save time hunting around modules
  • Teach new concepts in context of a larger ecosystem

Core

JavaScript and Node.js provide the central pillars.

The number one goal is to provide concrete examples for how this rapidly expanding micro-culture of web components can be pulled together compare and learn from different strategies.

Alternatives

It would be interesting see a completely different build engine and strategy using something like duo and graftjs for example.

Development Process

Client development - it starts up front: In rugby there is an adage that if you don’t win the battle upfront you will probably lose the game. By the same token if you don’t provide an engaging user experience you won’t win customers data. By starting upfront you don’t lose your story in the backend and create application that customer want to use. This approach is heavily influenced by the thoughts of Ryan Singer in The Art of Designing for People.

Services development - feed the backs: Again from rugby - though more controversial - you need to feed your backs to convert possesion and territory into points. This is analogous to capturing as much data as you can to turn into insights for growing profits.

Learning
